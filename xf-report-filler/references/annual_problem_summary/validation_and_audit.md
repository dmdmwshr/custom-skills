# 年度汇总校验与审计

## 数据源解析

审计读取年度根目录、候选底册、采用/未采用文件、逐条问题和输出路径。除显式 `--apply` 外，不写 Word 成品。

## 生成规则

- dry-run 输出 JSON 摘要，包含采用底册、未采用底册、问题清单、warning 和 blocker。
- `--review-json` 可写出逐条审计文件，用于人工核对。
- 正式生成前必须无 blocker。

## 不写入内容

- 不改产品底册源文件。
- 不改月度通报目录和 G01-G06 成品。
- 不把黄色问题从源底册中移除或改色。

## Warning/Blocker

- warning：同月多底册、黄色问题、未知大队、低置信度解析。
- blocker：无法识别月份、无大队分块、无法归属问题、Word 目录更新失败。

## 验收点

- 当前真实年度根目录 dry-run 不生成 Word。
- dry-run 审计能显示 2、3、4、5 月采用底册。
- review JSON 中保留原始问题全文，Word 正文只使用常见描述。
