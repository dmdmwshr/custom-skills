# 年度汇总校验与审计

## 数据源解析

审计读取年度根目录、候选底册、候选旧版网上巡查源、采用/未采用文件、逐条问题和输出路径。除显式 `--apply` 外，不写 Word 成品。

## 生成规则

- dry-run 输出 JSON 摘要，包含采用底册、未采用底册、问题清单、warning 和 blocker。
- 旧版网上巡查源会输出问题原文、年度正文描述、单独 ps 备注和段落号。
- `--review-json` 可写出逐条审计文件，用于人工核对。
- 正式生成前必须无 blocker。

## 不写入内容

- 不改产品底册源文件。
- 不改旧版网上巡查 `.doc` 源文件。
- 不改月度通报目录和 G01-G06 成品。
- 不把黄色问题从源底册中移除或改色。
- 不在 Word 正文生成可见目录页、TOC 字段、图片、扣分、扣分说明、扣分尾巴或条款号。
- 不把年度汇总的括号/ps 规则写入月度产品登记输出。

## Warning/Blocker

- warning：同月多源、黄色问题、未知大队、低置信度解析、旧版网上巡查问题需人工复核。
- blocker：无法识别月份、无大队分块、无法归属问题。

## 验收点

- 当前真实年度根目录 dry-run 不生成 Word。
- dry-run 审计能显示 1、2、3、4、5 月采用源。
- 1 月旧版网上巡查问题正文包含 `（ps：...）` 备注，但不包含扣分尾巴。
- 生成 Word 不含可见“目录”页和 TOC 字段；正文按“大队 -> 月份 -> 案卷 -> 问题”组织。
- 生成 Word 标题、正文、一级/二级标题字体字号符合 `../document_style.md`。
- 生成 Word 的封面标题水平居中、垂直居中；大队和月份标题、案卷和问题正文均为黑色。
- 黄色问题为黄底黑字，不得因高亮改变文字颜色。
- review JSON 中保留原始问题全文，Word 正文使用年度专用完整描述。
