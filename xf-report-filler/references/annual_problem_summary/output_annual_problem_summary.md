# 年度输出：产品监督底册问题汇总

## 文件用途

生成年度根目录下的内部复核文档：`YYYY年产品监督底册问题汇总.docx`。

## 数据源解析

- 依赖年度产品巡查底册解析结果。
- 按大队、月份、案卷聚合，再在案卷下列问题。
- 同一大队同一月份同一案卷只出现一次案卷信息；同一案卷下相同公开问题描述去重。
- 旧版网上巡查源的案卷信息使用项目编号和原始案卷信息行，不强行拆分立卷人。
- 年度汇总使用年度专用正文描述：保留底册括号内内容和旧版网上巡查 `ps` 备注，不套用月度公开材料的“括号前描述”规则。

## 生成规则

- 文档标题为 `YYYY年产品监督底册问题汇总`。
- 按 `../document_style.md` 的公文格式生成：标题 2 号方正小标宋，正文 3 号方正仿宋，一级标题 3 号方正黑体，二级标题 3 号方正楷体，页面按 22 行、每行 28 字网格设置，所有文字颜色为黑色。
- 只生成封面页和正文，不生成可见目录页，也不插入 TOC 字段。
- 封面标题水平居中、垂直居中，正文从下一页开始。
- 大队为一级标题，大队之间插入分页符。
- 月份为二级标题。
- 大队和月份标题用于 Word 左侧导航，但不得显示为蓝色或继承主题色。
- 案卷信息使用普通加粗正文，不作为标题层级，避免 Word 左侧导航过细。
- 案卷下按底册顺序列问题；同一案卷信息不在每条问题后重复。
- 问题正文保留括号内细节；半角括号统一为中文全角括号。
- 旧版网上巡查源列明原始案卷信息，不在正文标记“需人工复核”；独立 `ps` 段落追加到上一条问题后，格式为 `（ps：...）`。
- 大队顺序固定为梁溪、锡山、惠山、滨湖、新吴、江阴、宜兴、经开；未知大队排在最后。
- 黄色来源问题对应错误描述行继续设置黄色高亮。

## 不写入内容

- 不把扣分、扣分说明、分值、条款号写入正文。
- 不从源底册复制图片、截图或嵌入对象。
- 不生成 `.doc` 副本。
- dry-run 不写 Word 成品。

## Warning/Blocker

- 输出文件已存在且未传 `--force`：blocker。
- 存在无法归属问题：blocker。

## 验收点

- Word 中不存在可见“目录”页和 TOC 字段。
- 封面标题水平居中、垂直居中。
- 一级标题、二级标题、分页符齐全，左侧导航可按大队和月份跳转。
- 标题、正文、一级/二级标题字体字号符合公文格式，且颜色均为黑色。
- 同一案卷多个问题时，案卷信息只出现一次。
- 正文包含括号内细节和 `（ps：...）` 备注。
- 正文不出现扣分、扣分尾巴、条款号和图片。
- 黄色问题段落保留黄色高亮，文字仍为黑色。
- review JSON 与正文条目数量、来源文件和段落号可对应。
