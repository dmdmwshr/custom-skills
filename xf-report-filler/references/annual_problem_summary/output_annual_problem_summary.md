# 年度输出：产品监督底册问题汇总

## 文件用途

生成年度根目录下的内部复核文档：`YYYY年产品监督底册问题汇总.docx`。

## 数据源解析

- 依赖年度产品巡查底册解析结果。
- 按大队、月份、公开问题描述聚合。
- 同一大队同一月份同一错误描述只出现一次，案卷编号、题名、立卷人作为回查明细列出。
- 旧版网上巡查源的回查明细使用项目编号和原始案卷信息行，不强行拆分立卷人。

## 生成规则

- 文档标题为 `YYYY年产品监督底册问题汇总`。
- 标题下插入 Word 目录字段，正式生成时用 Word COM 更新目录。
- 大队为一级标题，大队之间插入分页符。
- 月份为二级标题。
- 错误描述下列明涉及案卷编号、题名、立卷人。
- 旧版网上巡查源列明编号和案卷信息，并标记需人工复核。
- 大队顺序固定为梁溪、锡山、惠山、滨湖、新吴、江阴、宜兴、经开；未知大队排在最后。
- 黄色来源问题对应错误描述行继续设置黄色高亮。

## 不写入内容

- 不把底册括号内、`ps` 后的字段级细节写入正文。
- 不把旧版网上巡查源的单独 `ps` 备注和扣分尾巴写入正文。
- 不生成 `.doc` 副本。
- dry-run 不写 Word 成品。

## Warning/Blocker

- Word COM 无法更新目录：blocker，删除临时成品，不留下半成品。
- 输出文件已存在且未传 `--force`：blocker。
- 存在无法归属问题：blocker。

## 验收点

- Word 中存在 TOC 字段。
- 一级标题、二级标题、分页符齐全。
- 黄色问题段落保留黄色高亮。
- review JSON 与正文条目数量、来源文件和段落号可对应。
