# 年度输出：产品监督底册问题汇总

## 文件用途

生成年度根目录下的内部复核文档：`YYYY年产品监督底册问题汇总.docx`。

## 数据源解析

- 依赖年度产品巡查底册解析结果。
- 按大队、月份、案卷聚合，再在案卷下列问题。
- 同一大队同一月份同一案卷只出现一次案卷信息；同一案卷下相同公开问题描述去重。
- 旧版网上巡查源的案卷信息使用项目编号和原始案卷信息行，不强行拆分立卷人。

## 生成规则

- 文档标题为 `YYYY年产品监督底册问题汇总`。
- 只生成封面页和正文，不生成可见目录页，也不插入 TOC 字段。
- 大队为一级标题，大队之间插入分页符。
- 月份为二级标题。
- 案卷信息使用普通加粗正文，不作为标题层级，避免 Word 左侧导航过细。
- 案卷下按底册顺序列问题；同一案卷信息不在每条问题后重复。
- 旧版网上巡查源列明原始案卷信息，不在正文标记“需人工复核”。
- 大队顺序固定为梁溪、锡山、惠山、滨湖、新吴、江阴、宜兴、经开；未知大队排在最后。
- 黄色来源问题对应错误描述行继续设置黄色高亮。

## 不写入内容

- 不把底册括号内、`ps` 后的字段级细节写入正文。
- 不把旧版网上巡查源的单独 `ps` 备注和扣分尾巴写入正文。
- 不把扣分、扣分说明、分值、条款号写入正文。
- 不从源底册复制图片、截图或嵌入对象。
- 不生成 `.doc` 副本。
- dry-run 不写 Word 成品。

## Warning/Blocker

- 输出文件已存在且未传 `--force`：blocker。
- 存在无法归属问题：blocker。

## 验收点

- Word 中不存在可见“目录”页和 TOC 字段。
- 一级标题、二级标题、分页符齐全，左侧导航可按大队和月份跳转。
- 同一案卷多个问题时，案卷信息只出现一次。
- 正文不出现扣分、`ps` 备注、括号内细节和图片。
- 黄色问题段落保留黄色高亮。
- review JSON 与正文条目数量、来源文件和段落号可对应。
