# 字段规则

## Excel 列映射

| 字段 | 列 | JSON key | 规则 |
| --- | --- | --- | --- |
| 单位名称 | B | `unit_name` | 取项目详情或检查记录表中的被检查单位名称。 |
| 项目编号 | C | `project_no` | 必填，用于去重更新。 |
| 单位地址 | D | `unit_address` | 取单位地址。 |
| 标称生产者 | E | `nominal_producers` | 取产品信息表中的“标称生产者”“标称生产厂”或同义字段，多个产品按产品顺序换行。 |
| 立卷人 | F | `case_handler` | 取主承办人，不取“承办人”中的协办人。 |
| 检查人 | G | `inspector` | 取审批人；如果审批人为“双随机、一公开”等非人名内容，留空。 |
| 检查时间 | H | `inspection_date` | 写入月份，如 `1月`。截图改名使用完整日期。 |
| 消防产品 | I | `products` | 多个产品换行写入；产品名和规格型号合并成一个字符串。 |
| 是否为微型消防站或快速处置队 | J | `station_or_team` | 根据产品所在部位判断。 |
| 现场判定或抽样送检 | K | `method` | 只允许 `现场判定` 或 `抽样送检`。 |
| 是否合格 | L | `qualified` | 只看初检；任一初检产品不合格则 `不合格`；全部初检明确合格才 `合格`；结果未出留空。 |
| 是否为行案/刑案 | M | `case_type` + `case_type_review` | 截图出现明确 `刑案` 才写 `刑案`；出现明显行政处罚文书号或 `罚字` 时默认写 `行案` 并设置 `case_type_review: true` 标黄；没有案件标志写 `否`。 |
| 是否为网售 | N | `online_sale` | 只有截图明确体现时写 `是` 或 `否`，否则留空。 |

## 下拉值

- 检查时间：`1月` 到 `12月`
- 是否为微型消防站或快速处置队：`微型消防站`、`快速处置队`、`否`
- 现场判定或抽样送检：`现场判定`、`抽样送检`
- 是否合格：`合格`、`不合格`
- 是否为行案/刑案：`刑案`、`行案`、`否`
- 是否为网售：`是`、`否`

## 截图分类

- `检查记录表`：标题或正文包含“消防产品监督检查记录”“产品所在部位”“监督检查时间”等内容。
- `案卷清单`：项目详情页、法律文书页、附件清单页、项目编号和基础信息列表。
- `其他附件`：不能归入以上两类但属于同一案的图片。

## 抽取约定

- 同一案件通常由重复命名的图片组成：数字组如 `1.png`、`11.png`、`111.png`，字母组如 `a.png`、`aa.png`、`aaa.png`。分组后仍要用项目编号或案卷号核对。
- 同一项目编号或案卷号出现初检、复检两套截图时，只收集初检。标题或结果中的 `[复查]`、`复检`、复查合格图标都不能覆盖初检结果，也不能新增一行。
- 立卷人以项目详情中的主承办人为准；检查记录表中“监督检查人员”可以作为辅助核对，不直接覆盖立卷人。
- 检查人以审批人为准，多个审批人时取最终审批人；明显不是人名的审批内容留空并在 `missing_fields` 中写 `inspector`。
- 标称生产者从“检查产品信息”“产品名称/规格型号”同一行的“标称生产者”列抽取。多个产品时，`nominal_producers` 与 `products` 保持同顺序；缺少某项时保留能确认的生产者，并在 `notes` 写明。
- 产品所在部位包含“微型消防站”时写 `微型消防站`；包含“快速处置队”时写 `快速处置队`；明确不是这两类时写 `否`。
- 抽样送检优先看检查记录或项目详情中的“抽样送检”标志；没有抽样送检且有现场检查/现场判定记录时写 `现场判定`。
- 初检标题包含 `[不合格]`，或初检产品检查结果为红叉/不合格，写 `不合格`。初检全部产品均为绿勾/合格时写 `合格`。只有复检结果或结果未出时，`qualified` 留空并在 `missing_fields` 写 `qualified`。
- 行案/刑案字段不要硬区分不确定案件。标题、文书号、附件或审批记录中出现明显 `罚字`、行政处罚告知/决定、罚款等行政处罚线索时，默认 `case_type` 写 `行案`，并设置 `case_type_review: true` 让脚本标黄。只有明确出现 `刑案`、刑事案件、移送公安等刑事线索时写 `刑案` 且不标黄。
- 网售字段不要推测。没有明确“网售”“网络销售”“电商平台”等信息时留空。
