# 字段规则

## Excel 列映射

| 字段 | 列 | JSON key | 规则 |
| --- | --- | --- | --- |
| 单位名称 | B | `unit_name` | 取项目详情或检查记录表中的被检查单位名称。 |
| 项目编号 | C | `project_no` | 必填，用于去重更新。 |
| 单位地址 | D | `unit_address` | 取单位地址。 |
| 立卷人 | E | `case_handler` | 取主承办人，不取“承办人”中的协办人。 |
| 检查人 | F | `inspector` | 取审批人；如果审批人为“双随机、一公开”等非人名内容，留空。 |
| 检查时间 | G | `inspection_date` | 写入月份，如 `1月`。截图改名使用完整日期。 |
| 消防产品 | H | `products` | 多个产品换行写入；产品名和规格型号合并成一个字符串。 |
| 是否为微型消防站或快速处置队 | I | `station_or_team` | 根据产品所在部位判断。 |
| 现场判定或抽样送检 | J | `method` | 只允许 `现场判定` 或 `抽样送检`。 |
| 是否合格 | K | `qualified` | 任一产品不合格则 `不合格`；全部明确合格才 `合格`；结果未出留空。 |
| 是否为行案/刑案 | L | `case_type` | 只有截图出现明确标志才写 `行案` 或 `刑案`，否则写 `否`。 |
| 是否为网售 | M | `online_sale` | 只有截图明确体现时写 `是` 或 `否`，否则留空。 |

## 下拉值

- G：`1月` 到 `12月`
- I：`微型消防站`、`快速处置队`、`否`
- J：`现场判定`、`抽样送检`
- K：`合格`、`不合格`
- L：`刑案`、`行案`、`否`
- M：`是`、`否`

## 截图分类

- `检查记录表`：标题或正文包含“消防产品监督检查记录”“产品所在部位”“监督检查时间”等内容。
- `案卷清单`：项目详情页、法律文书页、附件清单页、项目编号和基础信息列表。
- `其他附件`：不能归入以上两类但属于同一案的图片。

## 抽取约定

- 同一案件通常由相同数字重复命名的图片组成，例如 `1.png`、`11.png`。分组后仍要用项目编号核对。
- 立卷人以项目详情中的主承办人为准；检查记录表中“监督检查人员”可以作为辅助核对，不直接覆盖立卷人。
- 检查人以审批人为准，多个审批人时取最终审批人；明显不是人名的审批内容留空并在 `missing_fields` 中写 `inspector`。
- 产品所在部位包含“微型消防站”时写 `微型消防站`；包含“快速处置队”时写 `快速处置队`；明确不是这两类时写 `否`。
- 抽样送检优先看检查记录或项目详情中的“抽样送检”标志；没有抽样送检且有现场检查/现场判定记录时写 `现场判定`。
- 网售字段不要推测。没有明确“网售”“网络销售”“电商平台”等信息时留空。
