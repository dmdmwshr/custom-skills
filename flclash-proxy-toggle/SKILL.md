# FlClash Proxy Toggle

通过 FlClash 桌面窗口安全暂停或恢复代理。先检查当前状态，再执行单一步骤，最后回读验证。

## 硬规则

1. 代理处于运行状态时，禁止直接关闭 `FlClash` 应用程序。
2. 在代理仍运行、`系统代理` 仍开启或 `虚拟网卡` 仍开启时直接关闭应用，可能导致系统残留代理链路并出现没有网络的情况。
3. 用户想“关闭代理”时，优先点击右下角暂停/启动按钮，不要先关闭窗口。
4. 只有在已经暂停代理，或用户明确要求彻底退出且已确认当前状态安全时，才允许关闭应用窗口。
5. 不确定当前状态时，先检查，不猜测。

## 工具优先级

- 优先用桌面级只读工具确认状态：`list_apps`、`list_windows`、`take_ax_snapshot`、`take_screenshot`、`Snapshot`
- 执行操作时优先：聚焦窗口 -> 抓当前窗口截图 -> 单击
- 不依赖浏览器自动化

## 状态判断

组合判断状态，不要只看单一信号。

- 右下角大按钮左侧图标是 `||`：通常表示代理正在运行，点击后会暂停
- 右下角大按钮左侧图标是三角形：通常表示代理已暂停，点击后会恢复
- 右下角按钮计时器持续递增：通常表示运行中
- 右下角按钮计时器是 `00:00:00`：通常表示已暂停或刚进入暂停态
- `网络检测` 显示外网或代理出口 IP：通常表示代理在工作
- `网络检测` 变为直连出口 IP：通常表示已经退出代理链路
- 任务栏网络面板出现 `FlClash 已连接`：通常表示代理仍在运行

## 已知界面特性

1. 无障碍树可能只暴露右下角按钮的计时文本，不会直接暴露“暂停”或“启动”字样。
2. 判断按钮状态时，优先看截图中的图标形状，其次再看计时器和网络检测 IP。
3. 点击右下角按钮时，优先点击按钮左侧图标区域，不要优先点击右侧时间文本；时间文本区域可能不稳定或不触发状态切换。

## 工作流

### 暂停代理

1. 聚焦 `FlClash` 窗口。
2. 抓取当前窗口截图和无障碍树。
3. 确认右下角按钮当前是 `||` 图标，或确认计时器正在递增。
4. 点击右下角按钮左侧图标区域。
5. 立即回读状态，至少确认以下两项中的两项：
   - 图标变成三角形
   - 计时器变成 `00:00:00`
   - 网络速度归零
   - `网络检测` IP 切换为直连出口
   - 任务栏网络状态不再显示 `FlClash 已连接`
6. 如果状态没有变化，只允许再次单击同一按钮左侧图标区域。
7. 不要把“暂停代理”替换成“直接关闭应用”。

### 恢复代理

1. 聚焦 `FlClash` 窗口。
2. 抓取当前窗口截图和无障碍树。
3. 确认右下角按钮当前是三角形图标，或确认计时器是 `00:00:00`。
4. 点击同一按钮左侧图标区域恢复代理。
5. 立即回读状态，至少确认以下两项中的两项：
   - 图标变回 `||`
   - 计时器开始递增
   - `网络检测` 恢复为代理出口 IP
   - 任务栏网络状态重新出现 `FlClash 已连接`
6. 如果应用当前未运行，先启动 `FlClash`，再检查状态，不要假设“启动应用”就等于“代理已经恢复”。
7. 如果已经处于运行态，不要重复点击。

### 关闭应用

1. 只有在用户明确要求关闭 `FlClash` 应用时，才执行关闭窗口。
2. 执行关闭前，先确认代理已暂停，至少满足以下之一：
   - 右下角按钮是三角形图标
   - 右下角计时器为 `00:00:00`
   - 用户已经明确要求并且你已额外关闭相关代理链路并完成验证
3. 如果当前仍是 `||` 运行图标，先暂停代理，再考虑关闭窗口。
4. 如果无法确认当前是否已暂停，停止操作并向用户说明风险，不要强行关闭应用。

## 不要做的事

1. 不要在代理运行状态下直接关闭 `FlClash`。
2. 不要只因为窗口消失就认定代理已经关闭。
3. 不要只因为进程还在就认定代理仍然开启；必须结合按钮图标、计时器和网络状态一起判断。
4. 不要在状态模糊时连续乱点多个开关。
5. 不要把“暂停代理”和“关闭应用”混成一步。

## 输出约定

1. 先说明当前状态判断，再说明执行了什么动作。
2. 动作后必须回读验证，不要只报告“已点击”。
3. 如果因为安全规则拒绝直接关闭应用，要明确说明原因：代理运行时直接关掉 `FlClash` 可能导致没有网络。
